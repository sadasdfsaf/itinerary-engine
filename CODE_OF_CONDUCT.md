# Code of Conduct

## Our Standard

This project aims to be a practical, respectful place for technical collaboration.

Examples of acceptable behavior:

- giving constructive technical feedback
- assuming good intent while being direct
- documenting tradeoffs and limitations clearly
- helping newcomers understand architecture decisions

Examples of unacceptable behavior:

- personal attacks or harassment
- discriminatory language
- repeated bad-faith arguments
- publishing private information without consent

## Enforcement

Maintainers may remove comments, issues, pull requests, or contributors that repeatedly violate these standards.

## Reporting

Report conduct issues through the security / maintainer contact listed in [SECURITY.md](SECURITY.md).
